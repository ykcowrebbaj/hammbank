2）scala源码：

src/main/scala/com/kkb/spark/core/SparkCountCluster.scala: 

package com.kkb.spark.core

import org.apache.spark.rdd.RDD
import org.apache.spark.{SparkConf, SparkContext}

object SparkCountYarnCluster {
  def main(args: Array[String]): Unit = {
    val conf = new SparkConf().setAppName("WorkCount")
    val sc = new SparkContext(conf)
    val resultRDD: RDD[(String, Int)] = sc.textFile(args(0))
      .flatMap(_.split(" "))
      .map((_, 1))
      .reduceByKey(_ + _)
    resultRDD.saveAsTextFile(args(1))
    sc.stop()
  }

2、描述spark在yarn模式下，以cluster方式任务提交过程

  1. bin/spark-submit脚本启动，执行SparkSubmit里面的main方法
  2. 提交任务到ResourceManager(RM)
  3. 选择一台NodeManager(NM)启动Container，在该Container中运行ApplicationMaster(AM)进程，然后在AM中初始化Driver
  4. AM向RM申请划分Container的资源
  5. 申请到资源后，封装并发送指令 bin/java CoarseGrainedExecutorBackend，启动Container，创建ExcutorBackend，并在ExcutorBackend内部创建Executor
  6. Driver将分解后的Task发送到Executor进行执行
  7. 等待任务运行结束，SparkContext向RM申请注销，回收资源
  8. 任务执行完成，RM回收资源

3、填空题：

spark-submit提交应用时，如果需要用到额外的jar包，该使用参数( –jars )

–jars后面跟的参数是使用逗号隔开的列表，用以指定多个依赖的外部jar包

4、判断题：

spark standalone模式下，Driver跟Client处于同一个服务器上

错。Standalone-Client模式下，Driver和Client都处于本地主机；而Standalone-Cluster模式下Driver和Client是分离的，Client一般处于本地主机，Driver一般处于远程spark集群服务器的worker中。
