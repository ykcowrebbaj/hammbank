一、HBase表的数据模型有哪些？每个给出简要说明

HBase数据存储结构中表的数据模型主要包括：

行键（Row key）：table的主键，table中的记录按照rowkey 的字典序进行排序，在HBase存储系统中行键总是被看作一个Byte数组。

列族（Column Family）：HBase表中的每个列，都归属与某个列族。列族会在使用前定义（在定义表的时候就定义列族。

列限定符（Column Qualifier）：列限定符不需要提前定义（不需要在定义表和列族的时候就定义列限定符），在HBase存储系统中列限定符总是被看作一个Byte数组。

列（Column）：HBase 中的列由一个列族和一个列限定符组成，它们由=:=（冒号）字符分隔。

单元格（Cell）：根据行键、列族和列可以映射到一个对应的单元格，单元格是HBase存储数据的具体地址，。在单元格中存储具体数据值，它是以Byte数组的形式存储的。

时间戳：时间戳是给定"值"的一个版本号标识，每一个"值"都会对应一个时间戳。在默认情况下，时间戳表示数据服务在写入数据时的时间。

二、使用hbase命令行客户端，进行表的操作

（1）创建一个表，表名 "firsttable" ，包含两个列族，分别是col1 col2，其中col1设置可保留版本数为3，col2为1个： 

create 'firsttable', {NAME => 'col1', VERSIONS => 3}, {NAME => 'col2', VERSIONS => 1}

（2）向表中添加数据。要求：行键是rk001，向col1添加一列，列名为"oid", value是"order071701"

向col2添加一列，列名为"proid"，value是"pid00ab"，再向col1的列"oid"写入新的值，value是"order071702" 

put 'firsttable', 'rk001', 'col1:oid', 'order071701'
put 'firsttable', 'rk001', 'col2:proid', 'pid00ab'
put 'firsttable', 'rk001', 'col1:oid', 'order071702'

（3）查询firsttable表中，行键为rk001的行中的，col1列族下oid列的最新的两个版本的值 

get 'firsttable', 'rk001', {COLUMNS => 'col1:oid', VERSIONS => 2}
