远程桌面――ssh的X转发

在开始之前，为了避免混乱，我们先统一一下名称：

  • 本地主机：ssh客户端，亦是X服务端xorg-server，运行在windows操作系统上
  • 远程主机：ssh服务端，亦是X客户端firefox，运行于linux操作系统上

首先，下载cygwin的setup安装程序，http://www.cygwin.com/ （这里我使用的是32位的cygwin安装程序），然后安装会提示我们选择哪些程序，为了开启图形界面，必须安装以下这些包：

  • X11目录下的xorg-server（此项必须）
  • X11目录下的xinit （此项必须）
  • Net目录下的openssh （要支持ssh的X转发，也必须安装）

安装完之后，打开cygwin的shell，输入startxwin后，右下角会出现X托盘图标，这样，应该就算开启X11的服务了。然后打开本地主机上的xterm窗口（右键右下角X托盘图标，点击application –> xterm），在打开的xterm窗口中输入： 

ssh -Y root@remoteip

根据提示输入远程主机的登陆密码，最后启动firefox就应该ok了 

firefox &

但是，我在首次使用ssh的X转发时并不成功，便试着去掉&直接运行firefox，提示Error: no display specified。于是退出当前已登录的ssh，重新登陆并加上-v选项： 

ssh -v -Y root@remoteip

发现有如下提示： 

Warning: No xauth data; using fake authentication data for X11 forwarding.
debug1: Requesting X11 forwarding with authentication spoofing.
debug1: Remote: No xauth program; cannot forward with spoofing.
X11 forwarding request failed on channel 0

说明远程主机上还未安装xorg-x11-xauth包，于是： 

yum install xorg-x11-xauth

再重新ssh登陆运行firefox，稍等片刻windos桌面便弹出熟悉的firefox窗口，It works！:-)
